What is OpenClaw?

  • Local AI agent framework — runs on YOUR machine
  • Connects to Telegram, Signal, Discord, etc.
  • Always-on: memory, tools, and real actions
  • Think: chief of staff, not a chatbot
  • Why local: your data, your vault, your rules
A chatbot answers questions. A chief of staff gets things done.

Lessons Learned

  1. Start small — many cycles of tweaking
  2. Scripts for consistent output
  3. Programmatic enforcement > prompt guidance
  4. Self-diagnosis → self-correction → compound improvement
If your system can't refine itself based on what went wrong last week, you're just running the same broken process faster.

Hurdles

What Went Wrong
  1. "Done" ≠ actually done
    Logged a config change as complete without verifying. Change never applied. Crons routed wrong for 12+ hours.
  2. Sub-agent recreated a killed cron
    Deleted a cron. Sub-agent on a related task noticed it was missing and helpfully recreated it.
  3. Relayed output without verifying
    Reported "deploy successful" based on sub-agent claim. Deploy had actually failed.
How We're Catching It Earlier
  • 🧠 Memory system
    Every mistake logged to errors.md with WHAT/WHY/FIX/RULE. Agent reads it every session. Lessons accumulate across weeks.
  • 🎓 Gomez Training skill
    Periodic calibration: reviews context files, surfaces friction patterns, asks structured questions, makes file improvements. The agent trains itself.
  • ✅ Verification as a rule
    Sub-agent claims are untrusted. Every deploy, config change, and status report gets verified against the actual source before relaying.
Not just fixing problems after they happen — building systems to catch them before they do.

A Mistake I Made

Building too many systems at once — without giving each one enough time to iterate and actually work before moving on to the next.

The result: a storm of half-working automations, each depending on the last, none of them solid.

You end up debugging interactions between broken things instead of fixing one thing at a time.

The fix: ship one thing, run it for a week, make sure it's solid. Then build the next thing on top of it.
